### [\#](https://wpfusion.com/documentation/content-management/if-so/\#overview) Overview

WP Fusion integrates with [If-So Dynamic Content](https://wordpress.org/plugins/if-so/) to allow you to use tags or lists from [your connected CRM](https://wpfusion.com/documentation/faq/crm-compatibility-table/) as conditions in triggers and blocks.

### [\#](https://wpfusion.com/documentation/content-management/if-so/\#triggers) Triggers

When editing any If-So condition, select **CRM Tags (any)** from the _Select A Condition_ dropdown.

[![](https://wpfusion.com/wp-content/uploads/2021/08/if-so-1024x693.jpg)](https://wpfusion.com/wp-content/uploads/2021/08/if-so.jpg)

You can then select one or more CRM tags in the Select Tag(s) box, and add your content.

The content will then only be displayed to users who are logged in and have at least one of the specified tags.

### [\#](https://wpfusion.com/documentation/content-management/if-so/\#blocks) Blocks

When editing any block in the block editor, select _User has tag_ as a condition from the If-So conditions panel, and select a tag.

[![](data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%201024%20496'%3E%3C/svg%3E)](https://wpfusion.com/wp-content/uploads/2021/08/if-so-gutenberg.jpg)

The block will only be shown if the user is logged in and has the specified tag.
