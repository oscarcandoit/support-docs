## Description

The gform\_rule\_pre\_evaluation filters can be used to modify a conditional logic rule before it is evaulated.

## Usage

The gform\_rule\_pre\_evaluation filter has both a [JavaScript version](https://docs.gravityforms.com/gform_rule_pre_evaluation/#javascript-version) and a [PHP version](https://docs.gravityforms.com/gform_rule_pre_evaluation/#php-version). **Both versions should be used.**

The JavaScript version only overrides the conditional logic rule on the front-end.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5 | `gform.addFilter(``'gform_rule_pre_evaluation'``,``function``( rule, formId, conditionalLogic ) {`<br>```// Modify the rule.`<br>```return``rule;`<br>`} );` |

The PHP version overrides the conditional logic rule when it is evaulated during and post submission.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5 | `add_filter(``'gform_rule_pre_evaluation'``,``function``(``$rule``,``$form``,``$logic``,``$field_values``,``$entry``) {`<br>```// Modify the rule.`<br>```return``$rule``;`<br>`}, 10, 5 );` |

## JavaScript Version

### Parameters

- **rule** [Javascript Object](https://docs.gravityforms.com/javascript-object)
The current rule object. e.g.





|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5 | `{`<br>```"fieldId": "1",`<br>```"operator": "isnot",`<br>```"value": ""`<br>`}` |

- **formId** [integer](https://docs.gravityforms.com/integer) \| [string](https://docs.gravityforms.com/string)
The ID of the form in use.

- **conditionalLogic** [Javascript Object](https://docs.gravityforms.com/javascript-object)

The current conditional logic object. e.g.





|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9<br>10<br>11<br>12<br>13<br>14<br>15<br>16 | `{`<br>```"actionType": "show",`<br>```"logicType": "all",`<br>```"rules": [`<br>```{`<br>```"fieldId": "1",`<br>```"operator": "isnot",`<br>```"value": ""`<br>```},`<br>```{`<br>```"fieldId": "1",`<br>```"operator": "is",`<br>```"value": "{:1}"`<br>```}`<br>```]`<br>`}` |


### Example

This example shows how you can replace field merge tags used in the conditional logic rule value.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9<br>10 | `gform.addFilter(``'gform_rule_pre_evaluation'``,``function``( rule, formId ) {`<br>```var``mergeTags = GFMergeTag.parseMergeTags( rule.value );`<br>```if``( ! mergeTags.length ) {`<br>```return``rule;`<br>```}`<br>```rule.value = GFMergeTag.getMergeTagValue( formId, mergeTags[0][1], mergeTags[0][3] );`<br>```return``rule;`<br>`} );` |

### Placement

Your code snippet can be placed in a HTML field on your form or in a theme custom JavaScript file.

## PHP Version

### Parameters

- **$rule** [array](https://docs.gravityforms.com/array)
The current rule object. See the Rule Properties section of the [Conditional Logic Object](https://docs.gravityforms.com/conditional-logic/#rule-properties) article.

- **$form** [Form Object](https://docs.gravityforms.com/form-object)

The form currently being processed.

- **$logic** [Conditional Logic Object](https://docs.gravityforms.com/conditional-logic/)

The conditional logic.

- **$field\_values** [array](https://docs.gravityforms.com/array)

An array of dynamic population parameter keys with their corresponding values to be populated.

- **$entry** [Entry Object](https://docs.gravityforms.com/entry-object)

The entry currently being processed, if available.


### Example

This example shows how you can replace field merge tags used in the conditional logic rule value.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9<br>10<br>11<br>12<br>13 | `add_filter(``'gform_rule_pre_evaluation'``,``'filter_gform_rule_pre_evaluation'``, 10, 5 );`<br>`function``filter_gform_rule_pre_evaluation(``$rule``,``$form``,``$logic``,``$field_values``,``$entry``) {`<br>```if``( !``empty``(``$rule``[``'value'``] ) &&``strpos``(``$rule``[``'value'``],``'{'``) !== false ) {`<br>```remove_filter(``'gform_rule_pre_evaluation'``,``'filter_gform_rule_pre_evaluation'``);`<br>```if``(``empty``(``$entry``) ) {`<br>```$entry``= GFFormsModel::get_current_lead();`<br>```}`<br>```$rule``[``'value'``] = GFCommon::replace_variables(``$rule``[``'value'``],``$form``,``$entry``, false, false, false,``'text'``);`<br>```add_filter(``'gform_rule_pre_evaluation'``,``'filter_gform_rule_pre_evaluation'``, 10, 5 );`<br>```}`<br>```return``$rule``;`<br>`}` |

Note: In this example field merge tags are being processed before the entry has been saved so `GFFormsModel::get_current_lead()` is used to get a mock entry object for the current submission and `GFCommon::replace_variables()` is used to process the merge tags. Both of these methods can trigger the evaulation of conditional logic which will trigger this filter. To prevent a recursion related fatal error you must remove your hooked function before calling those methods and restore it after.

### Placement

Your code snippet should be placed in the [functions.php](https://codex.wordpress.org/Theme_Development#Functions_File) file of your active theme or a custom functions plugin.

### Since

This filter was added in Gravity Forms v2.4.22.

### Source Code

This filter is located in GFFormsModel::evaluate\_conditional\_logic() in _forms\_model.php_
