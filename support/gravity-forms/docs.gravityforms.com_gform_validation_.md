## Description

The “gform\_validation” filter allows custom validation logic to be used by Gravity Forms when a form is submitted.

## Usage

Applies to all forms.

```
add_filter( 'gform_validation', 'custom_validation', 10, 2 );
```

You can also specify this per form by adding the form id after the hook name.

```
add_filter( 'gform_validation_6', 'custom_validation', 10, 2 );
```

**Note**: DO NOT use the form specific version of this filter with forms that also have payment add-on feeds. If you need the validation to occur before the payment add-on validation use the generic version of the filter above and perform the form id check within the hooked function. Check the example below.

For an exhaustive walk-through on using the gform\_validation hook, see [Using the Gravity Forms “gform\_validation” Hook](https://docs.gravityforms.com/using-gform-validation-hook/).

## Parameters

- **$validation\_result** [array](https://docs.gravityforms.com/array) Contains the validation result and the current [Form Object](https://docs.gravityforms.com/form-object).

- **$context** [string](https://docs.gravityforms.com/string) The context for the current submission. Since Gravity Forms 2.6.3.2. Possible values: `form-submit`, `api-submit`, or `api-validate`.
