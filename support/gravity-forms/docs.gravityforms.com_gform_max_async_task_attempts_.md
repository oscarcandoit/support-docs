## Description

The `gform_max_async_task_attempts` filter allows developers to set the number of retries to be modified before the task is abandoned.

**Note:** This filter does not apply to the background feed processor. For that, use [gform\_max\_async\_feed\_attempts](https://docs.gravityforms.com/gform_max_async_feed_attempts/).

## Usage

|     |
| --- |
| `add_filter(``'gform_max_async_task_attempts'``,``'your_function_name'``, 10, 4 );` |

## Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| **$max\_attempts** | [int](https://docs.gravityforms.com/integer/) | The maximum number of attempts allowed. Default: `1`. |
| **$task** | mixed | The task about to be processed. |
| **$batch** | [object](https://docs.gravityforms.com/object/) | The batch currently being processed. |
| **$identifier** | [string](https://docs.gravityforms.com/string) | The string used to identify the type of background process. |
