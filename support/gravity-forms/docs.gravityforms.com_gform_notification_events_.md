## Description

Use this filter to add a new event to the drop down of available triggers for notifications. Additional code is necessary to handle the event since this hook only adds it to the Event drop down on the notifications admin page.

## Usage

|     |     |
| --- | --- |
| 1 | `add_filter(``'gform_notification_events'``,``'add_event'``, 10, 2);` |

## Parameters

| Parameter | Type | Description |
| --- | --- | --- |
| **$notification\_events** | [array](https://docs.gravityforms.com/array) | An array of notification events. |
| **$form** | [Form Object](https://docs.gravityforms.com/form-object) | The current Form object. |

## Examples

### 1\. User Registered Event

This example adds the trigger “User is registered” to the Event drop down on the Notifications page.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5 | `add_filter(``'gform_notification_events'``,``'add_event'``, 10, 2 );`<br>`function``add_event(``$notification_events``) {`<br>```$notification_events``[``'user_registered'``] = __(``'User is registered'``,``'gravityforms'``);`<br>```return``$notification_events``;`<br>`}` |

### 2\. Manual Notification Event

This example (by Gravity Wiz) adds the “Send Manually” notification event. This can be used in combination with the [Resend Notifications](https://docs.gravityforms.com/resend-notifications) feature to manually email users have submitted your form on demand. [Read the full tutorial for details.](http://gravitywiz.com/send-manual-notifications-with-gravity-forms/)

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5 | `add_filter(``'gform_notification_events'``,``'gw_add_manual_notification_event'``, 10, 2 );`<br>`function``gw_add_manual_notification_event(``$events``) {`<br>```$events``[``'manual'``] = __(``'Send Manually'``);`<br>```return``$events``;`<br>`}` |

### 3\. Payment Notification Events

See the [Send Notifications On Payment Events](https://docs.gravityforms.com/send-notifications-on-payment-events) article for code snippets for sending notifications when various payment related events occur.

### 4\. Mailchimp API failed Notification Event

This example adds the trigger “Mailchimp API Issue” to the Event drop down on the Notifications page for use in combination with the [Mailchimp API Issue custom action example](https://docs.gravityforms.com/gform_slug_error/#1-mailchimp-api-issue). You can adapt this example for use with any other Add-On Framework based add-on.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4 | `add_filter(``'gform_notification_events'``,``function``(``$events``) {`<br>```$events``[``'mailchimp_api_issue'``] =``'Mailchimp API Issue'``;`<br>```return``$events``;`<br>`} );` |

### 5\. Payment Cancelled event for Mollie

This example adds the trigger “Payment Cancelled” to the Event drop down on the Notifications page for forms having at least one feed to process payments via the Mollie add-on.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5<br>6<br>7 | `add_filter(``'gform_notification_events'``,``function``(``$notification_events``,``$form``) {`<br>```if``( function_exists(``'gf_mollie'``) && gf_mollie()->has_feed( rgar(``$form``,``'id'``) ) ) {`<br>```$notification_events``[``'void_authorization'``] =``'Payment Cancelled'``;`<br>```}`<br>```return``$notification_events``;`<br>`}, 10, 2 );` |

## Placement

This code can be used in the [functions.php](https://developer.wordpress.org/themes/basics/theme-functions/) file of the active theme, a custom functions plugin, a custom add-on, or with a code snippets plugin.

See also the PHP section in this article: [Where Do I Put This Code?](https://docs.gravityforms.com/where-do-i-put-this-code/)
