# Moderation Add-On Changelog

### 1.3.0 \| 2025-10-07

- Added the [`gform_moderation_score`](https://docs.gravityforms.com/gform_moderation_score/) filter, so the score can be overridden before it is saved to the entry meta.
- Added a notice to the form level settings when the user's Perspective API key is invalid/missing but custom toxic words are configured.
- Updated the settings page to list Perspective API settings in their own section.
- Updated the form settings page to allow custom words to be used without a Perspective API key.
- Fixed an issue where the search results for toxic entries are not displaying correctly in the Toxic Entries list.
- Fixed an issue where the Toxic Entries count is not updated correctly when an entry is marked as Spam or Trash on the entry detail page.
- Fixed an issue where global custom words are not applied.
- Removed the ability to star toxic entries.

### 1.2.0 \| 2024-10-30

- Added support for using the toxicity score with feed conditional logic. Requires Gravity Forms 2.9 or greater.
- Added the ability to manually set the moderation status of an entry.

### 1.1.0 \| 2024-09-05

- Fixed the inaccurate alert message when site language is not supported by Perspective.
- Fixed a PHP 8.1 deprecation notice that occurs on the entry detail page when an entry doesn't have a score.
- Fixed a PHP 8.1 deprecation notice that occurs on form submission if the custom toxic words setting is not configured.
- Fixed fatal error that can occur on the entry details page on sites running PHP 8.1.
