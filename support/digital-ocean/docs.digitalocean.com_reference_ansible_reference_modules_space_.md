# space

Generated on 25 Nov 2025
from `digitalocean.cloud` version
[`v1.2.1`](https://github.com/digitalocean/ansible-collection/releases/tag/v1.2.1)

## Synopsis

Manage Spaces. View the documentation at [https://www.digitalocean.com/products/spaces](https://www.digitalocean.com/products/spaces).

## Requirements

- pydo >= 0.1.3
- azure-core >= 1.26.1
- boto3 >= 1.28.53

## Parameters

| Parameter | Choices / Default | Description |
| --- | --- | --- |
| `aws_access_key_id`<br>str |  | The AWS\_ACCESS\_KEY\_ID to use for authentication. The `AWS_ACCESS_KEY_ID` environment variable will be used. |
| `aws_secret_access_key`<br>str |  | The AWS\_SECRET\_ACCESS\_KEY to use for authentication. The `AWS_SECRET_ACCESS_KEY` environment variable will be used. |
| `client_override_options`<br>dict |  | Client override options (developer use). For example, can be used to override the DigitalOcean API endpoint for an internal test suite. If provided, these options will knock out existing options. |
| `module_override_options`<br>dict |  | Module override options (developer use). Can be used to override module options to support experimental or future options. If provided, these options will knock out existing options. |
| `name`<br>str / **required** |  | The short name of the Space to create or delete. For example, given `name=my-do-space` and `region=nyc3`, the managed Space will be [https://my-do-space.nyc3.digitaloceanspace.com](https://my-do-space.nyc3.digitaloceanspace.com/). |
| `region`<br>str / **required** |  | The region in which to create or delete the Space. The `SPACES_REGION` environment variable will be used. |
| `state`<br>str | Choices:<br>- `present` (default)<br>- `absent` | State of the resource, `present` to create, `absent` to destroy. |
| `timeout`<br>int | Default: `300` | Polling timeout in seconds. |
| `token`<br>str |  | DigitalOcean API token. There are several environment variables which can be used to provide this value. `DIGITALOCEAN_ACCESS_TOKEN`, `DIGITALOCEAN_TOKEN`, `DO_API_TOKEN`, `DO_API_KEY`, `DO_OAUTH_TOKEN` and `OAUTH_TOKEN` |

## Examples

```yaml
- name: Create Space my-do-space in nyc3
  digitalocean.cloud.space:
    state: present
    name: my-do-space
    region: nyc3
    aws_access_key_id: "{{ aws_access_key_id}}"
    aws_secret_access_key: "{{ aws_secret_access_key}}"
```

## Return Values

| Key | Returned | Description |
| --- | --- | --- |
| `error`<br>dict | failure | DigitalOcean Spaces boto response metadata.<br>Sample:<br>```json<br>{<br>  "HTTPHeaders": "Header metadata key/values",<br>  "HTTPStatusCode": 400,<br>  "HostId": "Host ID data as a hash",<br>  "RequestId": "1234567890ABCDEF",<br>  "RetryAttempts'": 0<br>}<br>``` |
| `msg`<br>str | always | Space result information.<br>Sample:<br>```json<br>[<br>  "Created Space named my-do-space in nyc3",<br>  "Failed to create Space named my-do-space in nyc3",<br>  "No Spaces named my-do-space in nyc3, would create",<br>  "Existing Space named my-do-space in nyc3, would create",<br>  "Deleted Space named my-do-space in nyc3",<br>  "Failed to delete Space named my-do-space in nyc3",<br>  "Existing Space named my-do-space in nyc3, would delete",<br>  "Failed to list Spaces in nyc3"<br>]<br>``` |
| `spaces`<br>dict | always | Current spaces.<br>Sample:<br>```json<br>{<br>  "endpoint_url": "https://nyc3.digitaloceanspaces.com",<br>  "name": "my-do-space",<br>  "region": "nyc3",<br>  "space_url": "https://my-do-space.nyc3.digitaloceanspaces.com"<br>}<br>``` |
