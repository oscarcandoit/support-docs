Note:

Your [hosting provider](https://docs.cpanel.net/knowledge-base/cpanel-product/how-to-identify-your-hosting-provider/) can enable or disable this interface for resellers in WHM's [_Edit Reseller Nameservers and Privileges_](https://docs.cpanel.net/whm/resellers/edit-reseller-nameservers-and-privileges) interface ( _WHM >> Home >> Resellers >> Edit Reseller Nameservers and Privileges_).

Note:

This section does **not** appear on servers running the Ubuntu® operating system.

The _cPAddons Site Software_ section includes the following settings:

| Setting | Description | Values | Default |
| --- | --- | --- | --- |
| _cPAddons Site Software admin email_ | This setting specifies a contact email address that receives cPAddon moderation requests. The system also notifies resellers if they choose to configure their contact email address in the cPanel interface.<br>- A moderation request is a request from a user who wishes to install or upgrade a cPAddon. You **must** approve the request before the user installs or upgrades a cPAddon.<br>Note:<br>We have deprecated the moderation feature and will remove it in the future. You **cannot** enable moderation for any cPAddons. Any cPAddons that currently use moderation will continue to function, but if you disable this setting you **cannot** reactivate moderation. | To specify the cPAddon administrator’s email address, enter the email address that you wish to use in the text box. | _None_ |
| _cPAddons Site Software source automatic updates_ | This setting specifies whether you wish for cPanel & WHM to automatically update all of the cPAddons’ source files. | - _On_ — Enable<br>- _Off_ — Disable. | _On_ |
| _Max cPAddons Site Software installation requests_ | This setting specifies the maximum number of moderation requests that a single user can make at one time. | To specify a new value, enter the number of requests that you wish to allow in the text box.<br>- Minimum value: N/A<br>- Maximum value: N/A | `99` |
| _Max cPAddons Site Software installation requests per addon_ | This setting specifies the maximum number of moderation requests per cPAddon that a single user can make at one time. | To specify a new value, enter the number of requests that you wish to allow per cPAddon in the text box. <br>- Minimum value: N/A<br>- Maximum value: N/A | `99` |
| _cPAddons Site Software moderation notification_ | This setting allows you to select whether the cPAddons administrator receives notifications about pending moderation requests. | - _On_ — Enable<br>- _Off_ — Disable. | _Off_ |
| _Allow cPAddons Site Software installations from non-cPanel sources_ | This setting allows you to install third-party scripts on your server. | - _On_ — Enable<br>- _Off_ — Disable. | _On_ |
| _Allow cPAddons Site Software installations from modified sources_ | This setting allows users to install previously-altered cPAddons. You may wish to enable this item when you test custom cPaddons. | - _On_ — Enable<br>- _Off_ — Disable. | _Off_ |
| _Notify reseller of cPAddons Site Software installations_ | This setting notifies resellers whenever their users **must** update their cPAddons. | - _On_ — Enable<br>- _Off_ — Disable. | _On_ |
| _Notify root of cPAddons Site Software installations_ | This setting notifies the cPAddons Site Software administrator whenever their users **must** update their cPAddons. | - _On_ — Enable<br>- _Off_ — Disable. | _On_ |
| _Notify cPanel users when they need to update their cPAddons Site Software installations_ | This setting notifies cPanel users whenever they **must** update their cPAddons. | - _Allow users to choose_ — This option allows cPanel users to specify whether they wish to receive notifications about out-of-date cPAddons.<br>- _always_ — This option allows cPanel & WHM to automatically notify users when their cPAddons are out-of-date.<br>- _never_ — This option allows you to specify that users should **never** receive notifications when their cPAddons are out-of-date. | _Allow users to choose_ |
