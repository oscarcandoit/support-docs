# Collections

* * *

A collection is an group of vector records.
Records can be [added to or updated in](https://supabase.github.io/api.md#upserting-vectors) a collection.
Collections can be [queried](https://supabase.github.io/api.md#query) at any time, but should be [indexed](https://supabase.github.io/api.md#create-an-index) for scalable query performance.

Each vector record has the form:

```
1
2
3
4
5

Record (    id: String    vec: Numeric[]    metadata: JSON)
```

For example:

```
1

("vec1", [0.1, 0.2, 0.3], {"year": 1990})
```

Underneath every `vecs` collection is a Postgres table

```
1
2
3
4
5

create table <collection_name> (    id string primary key,    vec vector(<dimension>),    metadata jsonb)
```

where rows in the table map 1:1 with vecs vector records.

It is safe to select collection tables from outside the vecs client but issuing DDL is not recommended.
