# PayPal Standard Feed Meta

- [Introduction](https://docs.gravityforms.com/paypal-standard-feed-meta/#h-introduction)
- [Usage](https://docs.gravityforms.com/paypal-standard-feed-meta/#h-usage)

## Introduction

The [Feed Meta object](https://docs.gravityforms.com/feed-object) for the PayPal Standard Add-On is an associative array containing the properties which determine what type of transaction should occur and what values are sent to PayPal.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9<br>10 | `$feed``[``'meta'``] =``array``(`<br>```'feedName'``=>``'Your Feed Name'``,`<br>```'paypalEmail'``=>``'some@email.com'``,`<br>```'mode'``=>``'test'``,`<br>```'transactionType'``=>``'product'``,`<br>```'feed_condition_conditional_logic'``=> true,`<br>```'feed_condition_conditional_logic_object'``=>``array``(`<br>```'conditionalLogic'``=>``array``(),`<br>```),`<br>`);` |

## Usage

We recommend accessing the $feed meta using the rgar() or rgars() functions, e.g.:

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9<br>10<br>11<br>12<br>13 | `$conditional_logic_enabled``= rgars(``$feed``,``'meta/feedCondition'``);`<br>`if``(``$conditional_logic_enabled``) {`<br>```// Conditional logic is enabled for this feed.`<br>```$logic_rules``= rgars(``$feed``,``'meta/feed_condition_conditional_logic_object/conditionalLogic/rules'``);`<br>```foreach``(``$logic_rules``as``$rule``) {`<br>```// Process each conditional logic rule.`<br>```$field_id``= rgar(``$rule``,``'fieldId'``);`<br>```$operator``= rgar(``$rule``,``'operator'``);`<br>```$value``= rgar(``$rule``,``'value'``);`<br>```// Example: Use $field_id, $operator, and $value in your logic.`<br>```}`<br>`}` |

## Properties

- **feedName** [string](https://docs.gravityforms.com/string)
The feed name which appears on the add-ons feeds tab.

- **paypalEmail** [string](https://docs.gravityforms.com/string)

The PayPal email address where payment should be received.

- **mode** [string](https://docs.gravityforms.com/string)

Determines if the PayPal development sandbox should be used when testing or if live payments should be received. _Possible values: production or test._

- **transactionType** [string](https://docs.gravityforms.com/string)

The type of transaction which should occur when the feed is processed. _Possible values: product, subscription or donation._

- **billingInformation\_firstName** [string](https://docs.gravityforms.com/string)

The ID of the field containing the customer’s first name.

- **billingInformation\_lastName** [string](https://docs.gravityforms.com/string)

The ID of the field containing the customer’s last name.

- **billingInformation\_email** [string](https://docs.gravityforms.com/string)

The ID of the field containing the customer’s email.

- **billingInformation\_address** [string](https://docs.gravityforms.com/string)

The ID of the field containing the first line of the customer’s address.

- **billingInformation\_address2** [string](https://docs.gravityforms.com/string)

The ID of the field containing the second line of the customer’s address.

- **billingInformation\_city** [string](https://docs.gravityforms.com/string)

The ID of the field containing the city of the customer’s address.

- **billingInformation\_state** [string](https://docs.gravityforms.com/string)

The ID of the field containing the state of the customer’s address.

- **billingInformation\_zip** [string](https://docs.gravityforms.com/string)

The ID of the field containing the zip of the customer’s address.

- **billingInformation\_country** [string](https://docs.gravityforms.com/string)

The ID of the field containing the country of the customer’s address.

- **pageStyle** [string](https://docs.gravityforms.com/string)

Which PayPal page style should be used if you have setup a custom payment page style with PayPal.

- **continueText** [string](https://docs.gravityforms.com/string)

The text that should appear on the continue button once payment has been completed via PayPal.

- **cancelUrl** [string](https://docs.gravityforms.com/string)

The URL the user should be sent to should they cancel before completing their PayPal payment.

- **disableShipping** [boolean](https://docs.gravityforms.com/boolean)

Do not prompt buyer to include a shipping address. Default false.

- **disableNote** [boolean](https://docs.gravityforms.com/boolean)

Do not prompt buyer to include a note with payment. Default false.

- **delayNotification** [boolean](https://docs.gravityforms.com/boolean)

Send out selected notifications only after payment has been received. Default false.

- **selectedNotifications** [array](https://docs.gravityforms.com/array)

An array of notification IDs which should be delayed.

- **feed\_condition\_conditional\_logic** [boolean](https://docs.gravityforms.com/boolean)

Is the feed condition (conditional logic) setting enabled. Default is false.

- **feed\_condition\_conditional\_logic\_object** [array](https://docs.gravityforms.com/array)

An associative array containing the conditional logic rules. See the [Conditional Logic Object](https://docs.gravityforms.com/conditional-logic-object) for more details.


## Product or Donation Transaction Properties

The following properties are only applicable when the **transactionType** is set to **product** or **donation**.

- **paymentAmount** [string](https://docs.gravityforms.com/string)
What to use for the payment amount. _Possible values: form\_total or a product field ID._


## Subscription Transaction Properties

The following properties are only applicable when the **transactionType** is set to **subscription**.

- **recurringAmount** [string](https://docs.gravityforms.com/string)
What to use for the recurring amount. _Possible values: form\_total or a product field ID._

- **billingCycle\_length** [integer](https://docs.gravityforms.com/integer)

How often should the recurring payment occur. _Maximum values: 90 days, 52 weeks, 24 months or 5 years._

- **billingCycle\_unit** [string](https://docs.gravityforms.com/string)

What unit to use with the **billingCycle\_length**. _Possible values: day, week, month or year._

- **recurringTimes** [integer](https://docs.gravityforms.com/integer)

How many times the recurring payment should be made. The default (0) is to bill the customer until the subscription is canceled.

- **recurringRetry** [boolean](https://docs.gravityforms.com/boolean)

Try to bill again after failed attempt. Default is false.

- **trial\_enabled** [boolean](https://docs.gravityforms.com/boolean)

Should the recurring charge occur immediately or after a trial period? Default false.

- **trial\_product** [string](https://docs.gravityforms.com/string)

What is the source of the amount the user should be charged during the trial. _Possible values: enter\_amount or a product field ID._

- **trial\_amount** [string](https://docs.gravityforms.com/string)

The amount the user should be charged during the trial if **enter\_amount** is selected for the **trial\_product**.

- **trialPeriod\_length** [integer](https://docs.gravityforms.com/integer)

How long should the trial last. _Maximum values: 90 days, 52 weeks, 24 months or 5 years._

- **trialPeriod\_unit** [string](https://docs.gravityforms.com/string)

What unit to use with the **trialPeriod\_length**. _Possible values: day, week, month or year._


## Delayed Payment Properties

The following properties are only applicable when the “ _action_ only when payment is received.” options are being used.

- **type** [string](https://docs.gravityforms.com/string)
See **transactionType**. Only for backwards compatibility with non-framework add-ons.

- **recurring\_amount\_field** [string](https://docs.gravityforms.com/string)

See **recurringAmount**. Only for backwards compatibility with non-framework add-ons.

- **update\_user\_action** [string](https://docs.gravityforms.com/string)

User Registration add-on integration. What role should the user be assigned when the subscription is cancelled.

- **update\_site\_action** [string](https://docs.gravityforms.com/string)

User Registration add-on integration. Update site when the subscription is cancelled. _Possible values: deactivate or delete._

- **delay\_registration** [boolean](https://docs.gravityforms.com/boolean)

User Registration add-on integration. Register user only when payment is received.

- **delay\_gravityformsaweber** [boolean](https://docs.gravityforms.com/boolean)

Aweber add-on integration. Subscribe user to AWeber only when payment is received.

- **delay\_gravityformscampaignmonitor** [boolean](https://docs.gravityforms.com/boolean)

Campaign Monitor add-on integration. Subscribe user to Campaign Monitor only when payment is received.

- **delay\_gravityformsmailchimp** [boolean](https://docs.gravityforms.com/boolean)

MailChimp add-on integration. Subscribe user to MailChimp only when payment is received.

- **delay\_gravityformstwilio** [boolean](https://docs.gravityforms.com/boolean)

Twilio add-on integration. Send SMS only when a payment is received.

- **delay\_gravityformszapier** [boolean](https://docs.gravityforms.com/boolean)

Zapier add-on integration. Send feed to Zapier only when payment is received.
