## Description

This filter allows the webhook HTTP request data to be modified.

## Usage

The following would apply to all forms:

|     |     |
| --- | --- |
| 1 | `add_filter(``'gform_webhooks_request_data'``,``'your_function_name'``, 10, 4 )` |

To target a specific form, append the form id to the hook name. (format: gform\_webhooks\_request\_data\_FORMID)

|     |     |
| --- | --- |
| 1 | `add_filter(``'gform_webhooks_request_data_7'``,``'your_function_name'``, 10, 4 )` |

## Parameters

- **$request\_data** [array](https://docs.gravityforms.com/array)
HTTP request data.

- **$feed** [Feed Object](https://docs.gravityforms.com/feed-object)

The current feed object.

- **$entry** [Entry Object](https://docs.gravityforms.com/entry-object)

The current entry object.

- **$form** [Form Object](https://docs.gravityforms.com/form-object)

The current form object.

### 2\. Modify the field keys

This example shows how the field/input IDs, which are used as the keys to the data when the “all fields” choice is selected for the request body setting, can be replaced. In this case the period used in the input IDs is being replaced with an underscore so the webhook to Automate.io will succeed.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9<br>10<br>11<br>12<br>13<br>14 | `add_filter(``'gform_webhooks_request_data'``,``function``(``$request_data``,``$feed``) {`<br>```if``( rgars(``$feed``,``'meta/requestBodyType'``) ===``'all_fields'`<br>```&&``strpos``( rgars(``$feed``,``'meta/requestURL'``),``'automate.io'``) !== false`<br>```) {`<br>```foreach``(``$request_data``as``$key``=>``$value``) {`<br>```if``(``is_numeric``(``$key``) ) {`<br>```$request_data``[``'field '``.``str_replace``(``'.'``,``'_'``,``$key``) ] =``$value``;`<br>```unset(``$request_data``[``$key``] );`<br>```}`<br>```}`<br>```}`<br>```return``$request_data``;`<br>`}, 10, 2 );` |

### 3\. Modify the field values for the Events Calendar plugin

This example shows how some field values, which will be strings when mapped to form field values, can be changed to booleans as required by the Events Calendar plugin.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9<br>10<br>11<br>12<br>13<br>14<br>15<br>16<br>17<br>18<br>19<br>20<br>21 | `add_filter(``'gform_webhooks_request_data'``,``function``(``$request_data``,``$feed``) {`<br>```$request_url``= rgars(``$feed``,``'meta/requestURL'``);`<br>```if``(``strpos``(``$request_url``,``'tribe/events/'``) !== false ) {`<br>```$requires_bool_values``=``array``(`<br>```'all_day'``,`<br>```'show_map'``,`<br>```'show_map_link'``,`<br>```'hide_from_listings'``,`<br>```'sticky'``,`<br>```'featured'``,`<br>```);`<br>```foreach``(``$request_data``as``$key``=>``$value``) {`<br>```if``( in_array(``$key``,``$requires_bool_values``) ) {`<br>```$request_data``[``$key``] = (bool)``$value``;`<br>```}`<br>```}`<br>```}`<br>```return``$request_data``;`<br>`}, 10, 2 );` |

This example shows how empty fields can be removed from the request data when the “all fields” choice is selected for the request body setting.
