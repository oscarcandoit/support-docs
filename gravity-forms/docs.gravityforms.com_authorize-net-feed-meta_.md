# Authorize.Net Feed Meta

- [Introduction](https://docs.gravityforms.com/authorize-net-feed-meta/#h-introduction)
- [Usage](https://docs.gravityforms.com/authorize-net-feed-meta/#h-usage)

## Introduction

The [Feed Meta object](https://docs.gravityforms.com/feed-object) for the Authorize.Net Add-On is an associative array containing the properties which determine what type of transaction should occur and what values are sent to Authorize.Net.

```
$feed['meta'] = array(
    'feedName' => 'Your Feed Name',
    'transactionType' => 'product',
    'feed_condition_conditional_logic' => true,
    'feed_condition_conditional_logic_object' => array(
        'conditionalLogic' => array(),
    ),
);
```

## Usage

We recommend accessing the $feed meta using the rgar() or rgars() functions, e.g.:

```
$conditional_logic_enabled = rgars( $feed, 'meta/feedCondition' );

if ( $conditional_logic_enabled ) {
    // Conditional logic is enabled for this feed.
    $logic_rules = rgars( $feed, 'meta/feed_condition_conditional_logic_object/conditionalLogic/rules' );
    foreach ( $logic_rules as $rule ) {
        // Process each conditional logic rule.
        $field_id  = rgar( $rule, 'fieldId' );
        $operator  = rgar( $rule, 'operator' );
        $value     = rgar( $rule, 'value' );
        // Example: Use $field_id, $operator, and $value in your logic.
    }
}
```

## Properties

- **feedName** [string](https://docs.gravityforms.com/string)
The feed name which appears on the add-ons feeds tab.

- **transactionType** [string](https://docs.gravityforms.com/string)

The type of transaction which should occur when the feed is processed. _Possible values: product or subscription._

- **billingInformation\_email** [string](https://docs.gravityforms.com/string)

The ID of the field containing the customer’s email.

- **billingInformation\_address** [string](https://docs.gravityforms.com/string)

The ID of the field containing the first line of the customer’s address.

- **billingInformation\_address2** [string](https://docs.gravityforms.com/string)

The ID of the field containing the second line of the customer’s address.

- **billingInformation\_city** [string](https://docs.gravityforms.com/string)

The ID of the field containing the city of the customer’s address.

- **billingInformation\_state** [string](https://docs.gravityforms.com/string)

The ID of the field containing the state of the customer’s address.

- **billingInformation\_zip** [string](https://docs.gravityforms.com/string)

The ID of the field containing the zip of the customer’s address.

- **billingInformation\_country** [string](https://docs.gravityforms.com/string)

The ID of the field containing the country of the customer’s address.

- **enableReceipt** [boolean](https://docs.gravityforms.com/boolean)

Send Authorize.Net email receipt? Default is false.

- **apiSettingsEnabled** [boolean](https://docs.gravityforms.com/boolean)

Override the default API settings? Default is false.

- **overrideMode** [string](https://docs.gravityforms.com/string)

Which mode should this feed use when overriding the default API settings. _Possible values: production or test._

- **overrideLogin** [string](https://docs.gravityforms.com/string)

The API login ID to use when overriding the default API settings.

- **overrideKey** [string](https://docs.gravityforms.com/string)

The API transaction key to use when overriding the default API settings.

- **feed\_condition\_conditional\_logic** [boolean](https://docs.gravityforms.com/boolean)

Is the feed condition (conditional logic) setting enabled. Default is false.

- **feed\_condition\_conditional\_logic\_object** [array](https://docs.gravityforms.com/array)

An associative array containing the conditional logic rules. See the [Conditional Logic Object](https://docs.gravityforms.com/conditional-logic-object) for more details.


## Product Transaction Properties

The following properties are only applicable when the **transactionType** is set to **product**.

- **paymentAmount** [string](https://docs.gravityforms.com/string)
What to use for the payment amount. _Possible values: form\_total or a product field ID_


## Subscription Transaction Properties

The following properties are only applicable when the **transactionType** is set to **subscription**.

- **recurringAmount** [string](https://docs.gravityforms.com/string)
What to use for the recurring amount. _Possible values: form\_total or a product field ID_

- **billingCycle\_length** [integer](https://docs.gravityforms.com/integer)

How often should the recurring payment occur. _Minimum values: 7 days or 1 month_. _Maximum values: 365 days or 12 months._

- **billingCycle\_unit** [string](https://docs.gravityforms.com/string)

What unit to use with the **billingCycle\_length**. _Possible values: day or month._

- **recurringTimes** [integer](https://docs.gravityforms.com/integer)

How many times the recurring payment should be made. The default (0) is to bill the customer until the subscription is canceled.

- **setupFee\_enabled** [boolean](https://docs.gravityforms.com/boolean)

Should a setup fee be charged? Default is false.

- **setupFee\_product** [string](https://docs.gravityforms.com/string)

The ID of the product field containing the amount the user should be charged for the setup fee.

- **trial\_enabled** [boolean](https://docs.gravityforms.com/boolean)

Should the recurring charge occur immediately or after a trial period? Default false.

- **trial\_product** [string](https://docs.gravityforms.com/string)

What is the source of the amount the user should be charged during the trial. _Possible values: enter\_amount or a product field ID._

- **trial\_amount** [string](https://docs.gravityforms.com/string)

The amount the user should be charged during the trial if **enter\_amount** is selected for the **trial\_product**.

- **update\_post\_action** [string](https://docs.gravityforms.com/string)

Change the post status when a subscription is canceled? _Possible values: draft or delete._
