## Description

The gform\_pre\_render filter is executed before the form is displayed and can be used to manipulate the [Form Object](https://docs.gravityforms.com/form-object) before rendering the form.

This filter should be used in conjunction with the [gform\_pre\_validation](https://docs.gravityforms.com/gform_pre_validation), [gform\_pre\_submission\_filter](https://docs.gravityforms.com/gform_pre_submission_filter), and [gform\_admin\_pre\_render](https://docs.gravityforms.com/gform_admin_pre_render) filters to update the [Form Object](https://docs.gravityforms.com/form-object) to be able to use those values elsewhere (merge tags in the confirmation and notification, for example).

**IMPORTANT**: The **Dynamic** Population feature uses **PHP** to get the value(s) and populate the form field(s). Therefore, it can’t be used in cached pages. This is not a Gravity Forms limitation but a consequence of using caching, which prevents PHP code from running.

## Usage

The following would apply to all forms.

|     |
| --- |
| `add_filter(``'gform_pre_render'``,``'your_function_name'``);` |

To limit the scope of your function to a specific form, append the form id to the end of the hook name. (format: gform\_pre\_render\_FORMID)

|     |
| --- |
| `add_filter(``'gform_pre_render_6'``,``'your_function_name'``);` |

## Parameters

- **$form** [array](https://docs.gravityforms.com/array) \| [bool](https://docs.gravityforms.com/bool)

The [form object](https://docs.gravityforms.com/form-object/) of the form to be displayed or `false` to display the form not found message.
- **$ajax** [bool](https://docs.gravityforms.com/bool)

Is AJAX enabled. Only used when `$context` is `form_display`.
- $ **field\_values** [array](https://docs.gravityforms.com/array)

An array of dynamic population parameter keys with their corresponding values to be populated. Only used when `$context` is `form_display`.
- **$context** [string](https://docs.gravityforms.com/string/)

The context that the method is being called in. Possible values are `form_display` and `form_config`.
