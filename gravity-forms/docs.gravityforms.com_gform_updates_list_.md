## Description

The “gform\_updates\_list” filters the available updates displayed on the Forms -> System Status -> Updates page.

## Usage

**Note:** Set the run priority higher than the default of 10 because this filter is also added in the auto upgrade Gravity Forms code and the list of updates will not be complete unless the filter in the GF code runs before your added filter.

|     |     |
| --- | --- |
| 1 | `add_filter(``'gform_updates_list'``,``'your_function_name'``, 11, 1 );` |

## Parameters

- **$updates** [array](https://docs.gravityforms.com/array)
An array of plugins displayed on the Updates page.


## Example

This example removes a plugin from the list of available updates.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9<br>10<br>11<br>12<br>13<br>14<br>15<br>16 | `add_filter(``'gform_updates_list'``,``'remove_update'``, 11, 1 );`<br>`function``remove_update(``$updates``){`<br>```//remove user registration from the updates list`<br>```foreach``(``$updates``as``$id``=>``$update``){`<br>```//loop through array and create ids for the sub arrays`<br>```//so the user registration plugin can be removed`<br>```if``(``$update``[``'slug'``] ==``'gravityformsuserregistration'``){`<br>```//remove it from the main array`<br>```unset(``$updates``[``$id``] );`<br>```break``;`<br>```}`<br>```}`<br>```return``$updates``;`<br>`}` |

## Since

This filter was added in v2.2.

## Placement

This code should be placed in the [functions.php](http://codex.wordpress.org/Theme_Development#Functions_File) file of your active theme.

## Source Code

This filter is located in GF\_Update::available\_updates() in _gravityforms/includes/system-status/class-gf-update.php_.
