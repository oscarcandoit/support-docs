## Description

This filter can be used to delay feed processing for [Add-On Framework](https://docs.gravityforms.com/category/developers/php-api/add-on-framework) based add-ons.

## Usage

The base filter which would run for all forms and all feeds would be used like so:

|     |
| --- |
| `add_filter(``'gform_is_delayed_pre_process_feed'``,``'your_function_name'``, 10, 4 );` |

To target a specific form append the form id to the hook name. (format: gform\_is\_delayed\_pre\_process\_feed\_FORMID)

|     |
| --- |
| `add_filter(``'gform_is_delayed_pre_process_feed_10'``,``'your_function_name'``, 10, 4 );` |

## Parameters

- **$is\_delayed** [boolean](https://docs.gravityforms.com/boolean)
Is feed processing delayed?

- **$form** [Form Object](https://docs.gravityforms.com/form-object)

The form currently being processed.

- **$entry** [Entry Object](https://docs.gravityforms.com/entry-object)

The entry currently being processed.

- **$slug** [string](https://docs.gravityforms.com/string)

The add-on slug, including the **gravityforms** prefix. See the [Gravity Forms Add-On Slugs](https://docs.gravityforms.com/gravity-forms-add-on-slugs) article for a list of possible slugs.

If you don’t, the feed would remain delayed when you trigger feed processing after your event has occurred.
