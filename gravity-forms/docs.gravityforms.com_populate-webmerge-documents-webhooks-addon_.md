## Introduction

Using the Gravity Forms Webhooks Add-On it is possible to populate [WebMerge](https://www.webmerge.me/) documents when a form is submitted.

**Note:** This article assumes you have [installed and activated](https://docs.gravityforms.com/how-to-install-gravity-forms-add-ons/) the Webhooks Add-On and have created a document in your [WebMerge account](https://www.webmerge.me/manage/). If not, you will need to do so before any of the functionality in this article can be used.

## Configuring the Webhook

To configure webhooks for a form go to your **Form Settings** and click on the **Webhooks** tab and then click the **Add New** button.

Next, you’ll need to configure the request that is being made.

[![WebMerge Webhook Feed Settings](https://docs.gravityforms.com/wp-content/uploads/2021/07/CleanShot-2021-07-10-at-18.54.04.png)](https://docs.gravityforms.com/wp-content/uploads/2021/07/CleanShot-2021-07-10-at-18.54.04.png)

#### Name

Enter a name for your feed. This can be anything you want as it is only used for identification purposes on the feeds list page and in the add-on log.

#### Request URL

Enter the **Merge URL** from your WebMerge documents Merge tab. e.g. **https://www.webmerge.me/merge/12345/abcde**

When performing a test request append the **test=1** query string parameter to the URL so the merge does not count against your accounts merge credits e.g. **https://www.webmerge.me/merge/12345/abcde?test=1**

#### Request Method

Select the **POST** choice as the request method.

#### Request Format

Select the **JSON** choice as the request format.

#### Request Headers

There are no request headers to configure when integrating with WebMerge.

[![WebMerge Webhook Feed Settings - Request Headers](https://docs.gravityforms.com/wp-content/uploads/2021/07/image-25.png)](https://docs.gravityforms.com/wp-content/uploads/2021/07/image-25.png)

#### Request Body

Select the **Select Fields** choice.

#### Field Values

Configure rows for each of your documents merge fields. Click the add item icon (+) to add new rows.

You can find the keys to use on your WebMerge documents Test tab, they are the labels above each test value input, ignoring the JSON merge field at the end.

[![](https://docs.gravityforms.com/wp-content/uploads/2018/04/webmerge-test-merge-fields-300x91.png)](https://docs.gravityforms.com/wp-content/uploads/2018/04/webmerge-test-merge-fields.png)

For the values you can select from the form fields, entry meta, or **Add Custom Value** which will display a text input which also supports [Merge Tags](https://docs.gravityforms.com/category/user-guides/merge-tags-getting-started/).

#### Webhook Condition

If WebMerge document population should be conditional based on the value of a form field select the **Enable Condition** checkbox and then configure the rules which should be met for this request to be sent.

## Alternative Method

If you don’t have an Elite or legacy Developer license and would prefer not to [upgrade your existing license](https://docs.gravityforms.com/licensing-faq/#license-upgrades), you can still populate WebMerge documents by using one of the following methods:

- Pro license holders can use the Zapier Add-On. See the [WebMerge Gravity Forms support article](https://support.webmerge.me/hc/en-us/articles/206526246-GravityForms) for an example.
- Anyone can use the [gform\_after\_submission](https://docs.gravityforms.com/gform_after_submission) filter in the theme functions.php file or a custom functionality plugin. See the [Populate a Living Will from Gravity Forms](https://www.webmerge.me/blog/populate-a-living-will-from-gravity-forms) article by WebMerge for more details.
