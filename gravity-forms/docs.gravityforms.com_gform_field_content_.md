## Description

This filter is executed before creating the field’s content, allowing users to completely modify the way the field is rendered. It can also be used to create custom field types.

It is similar to [gform\_field\_input](https://docs.gravityforms.com/gform_field_input), but provides more flexibility.

## Usage

```
add_filter( 'gform_field_content', 'my_custom_function', 10, 5 );
```

```
// apply to a specific form
add_filter( 'gform_field_content_123', 'my_custom_function', 10, 5 );
```

```
// apply to a specific form and field
add_filter( 'gform_field_content_123_6', 'my_custom_function', 10, 5 );
```

## Parameters

- **$field\_content** [string](https://docs.gravityforms.com/string)
The field content to be filtered.

- **$field** [Field Object](https://docs.gravityforms.com/field-object)

The field that this input tag applies to.

- **$value** [string](https://docs.gravityforms.com/string)

The default/initial value that the field should be pre-populated with.

- **$entry\_id** [integer](https://docs.gravityforms.com/integer)

When executed from the entry detail screen, $lead\_id will be populated with the Entry ID. Otherwise, it will be 0.

- **$form\_id** [integer](https://docs.gravityforms.com/integer)

The current Form ID.

## Placement

This code should be placed in the [functions.php file](https://codex.wordpress.org/Theme_Development#Functions_File) of your active theme.

- GFEntryDetail::lead\_detail\_grid() in _entry\_detail.php_
