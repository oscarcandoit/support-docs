# Stripe Feed Meta

## Introduction

The [Feed Meta object](https://docs.gravityforms.com/feed-object) for the Stripe Add-On is an associative array containing the properties which determine what type of transaction should occur and what values are sent to Stripe.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9<br>10 | `$feed``[``'meta'``] =``array``(`<br>```'feedName'``=>``'Your Feed Name'``,`<br>```'transactionType'``=>``'product'``,`<br>```'paymentAmount'``=>``'form_total'``,`<br>```'receipt_field'``=>``'3'``,`<br>```'feed_condition_conditional_logic'``=> true,`<br>```'feed_condition_conditional_logic_object'``=>``array``(`<br>```'conditionalLogic'``=>``array``(),`<br>```),`<br>`);` |

## Usage

We recommend accessing the $feed meta using the rgar() or rgars() functions, e.g.:

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5<br>6<br>7<br>8<br>9<br>10<br>11<br>12<br>13 | `$conditional_logic_enabled``= rgars(``$feed``,``'meta/feedCondition'``);`<br>`if``(``$conditional_logic_enabled``) {`<br>```// Conditional logic is enabled for this feed.`<br>```$logic_rules``= rgars(``$feed``,``'meta/feed_condition_conditional_logic_object/conditionalLogic/rules'``);`<br>```foreach``(``$logic_rules``as``$rule``) {`<br>```// Process each conditional logic rule.`<br>```$field_id``= rgar(``$rule``,``'fieldId'``);`<br>```$operator``= rgar(``$rule``,``'operator'``);`<br>```$value``= rgar(``$rule``,``'value'``);`<br>```// Example: Use $field_id, $operator, and $value in your logic.`<br>```}`<br>`}` |

## Properties

- **feedName** [string](https://docs.gravityforms.com/string)
The feed name which appears on the add-ons feeds tab.

- **transactionType** [string](https://docs.gravityforms.com/string)

The type of transaction which should occur when the feed is processed. _Possible values: product or subscription._

- **metaData** [array](https://docs.gravityforms.com/array)

An indexed array of custom meta information. Stripe allows a maximum of 20 custom keys to be sent. Each custom meta array contains the following properties.


|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5 | `array``(`<br>```'key'``=>``'gf_custom'``,`<br>```'value'``=>``'1.3'``,`<br>```'custom_key'``=>``'product_quantity'``,`<br>`);` |

- **key** [string](https://docs.gravityforms.com/string)


Value should be gf\_custom. Instructs the dynamic\_field\_map setting to use the custom\_key input.

- **value** **[string](https://docs.gravityforms.com/string)**

The ID of the field containing the value to be sent for the specified custom\_key. The mapped data will be truncated to 500 characters per requirements by Stripe.
- **custom\_key** [string](https://docs.gravityforms.com/string)

The custom key you want to use when sending the custom metadata to Stripe. Must be 40 characters or less.

- **feed\_condition\_conditional\_logic** [boolean](https://docs.gravityforms.com/boolean)

Is the feed condition (conditional logic) setting enabled. Default is false.
- **feed\_condition\_conditional\_logic\_object** [array](https://docs.gravityforms.com/array)

An associative array containing the conditional logic rules. See the [Conditional Logic Object](https://docs.gravityforms.com/conditional-logic-object) for more details.

## Product Transaction Properties

The following properties are only applicable when the **transactionType** is set to **product**.

- **paymentAmount** [string](https://docs.gravityforms.com/string)


What to use for the payment amount. _Possible values: form\_total or a product field ID._

- **receipt\_field** [string](https://docs.gravityforms.com/string) The ID of the field containing the email Stripe should use when sending their receipt.


## Subscription Transaction Properties

The following properties are only applicable when the **transactionType** is set to **subscription**.

- **recurringAmount** [string](https://docs.gravityforms.com/string)


What to use for the recurring amount. _Possible values: form\_total or a product field ID_

- **billingCycle\_length** [integer](https://docs.gravityforms.com/integer) How often should the recurring payment occur. _Maximum values: 1 year or 12 weeks or 12 months._

- **billingCycle\_unit** [string](https://docs.gravityforms.com/string) How often should the recurring payment occur. _Possible values: week, month or year._

- **setupFee\_enabled** [boolean](https://docs.gravityforms.com/boolean) Should a setup fee be charged? Default is false.

- **setupFee\_product** [string](https://docs.gravityforms.com/string) The ID of the product field containing the amount the user should be charged for the setup fee.

- **trial\_enabled** [boolean](https://docs.gravityforms.com/boolean) Should the recurring charge occur immediately or after a trial period? Default false.

- **trialPeriod** [integer](https://docs.gravityforms.com/integer) The number of days the trial period should last.

- **customerInformation\_email** [string](https://docs.gravityforms.com/string) The ID of the field containing the customer email.

- **customerInformation\_description** [string](https://docs.gravityforms.com/string) The ID of the field containing the customer description.
