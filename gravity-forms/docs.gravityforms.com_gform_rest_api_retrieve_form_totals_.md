## Description

The gform\_rest\_api\_retrieve\_form\_totals filter allows the entries property (entry totals) to be omitted from the GET /forms (REST API v2) response. This is useful for increasing performance of the endpoint when totals aren’t required.

## Usage

The filter would be used like so:

|     |     |
| --- | --- |
| 1 | `add_filter(``'gform_rest_api_retrieve_form_totals'``,``'your_function_name'``, 10, 2 );` |

You can also specify this per form by adding the form id after the filter name.

|     |     |
| --- | --- |
| 1 | `add_filter(``'gform_rest_api_retrieve_form_totals_6'``,``'your_function_name'``, 10, 2 );` |

## Parameters

- **$include\_totals** [boolean](https://docs.gravityforms.com/boolean)
Whether to include totals; defaults to true.

- **$form** [object](https://docs.gravityforms.com/object)

The current form properties (id, title, date\_created, is\_active).


## Examples

### 1\. Simple Usage

|     |     |
| --- | --- |
| 1 | `add_filter(``'gform_rest_api_retrieve_form_totals'``,``'__return_false'``);` |

### 2\. Advanced Usage

|     |     |
| --- | --- |
| 1<br>2<br>3 | `add_filter(``'gform_rest_api_retrieve_form_totals'``,``function``(``$include_totals``,``$form``) {`<br>```return``! in_array(``$form``->id,``array``( 1, 3, 5 ) );`<br>`}, 10, 2 );` |

## Placement

This code should be placed in the [functions.php](http://codex.wordpress.org/Theme_Development#Functions_File) file of your active theme or a custom functions plugin.

## Since

This filter was added in Gravity Forms v2.4.24.1.

## Source Code

This filter is located in _GF\_REST\_Forms\_Controller::get\_items_ in _includes/webapi/v2/includes/controllers/class-controller-forms.php_.
