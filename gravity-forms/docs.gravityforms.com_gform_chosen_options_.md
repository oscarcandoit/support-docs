- [Description](https://docs.gravityforms.com/gform_chosen_options/#h-description)
- [Usage](https://docs.gravityforms.com/gform_chosen_options/#h-usage)

- [Source Code](https://docs.gravityforms.com/gform_chosen_options/#h-source-code)

## Description

Allows the [Chosen](http://harvesthq.github.com/chosen/) jQuery script settings for Drop Down and Multi-Select fields to be modified when the “Enable enhanced user interface” is checked. Fires when the form is loaded and has a field using Chosen.

## Usage

|     |     |
| --- | --- |
| 1<br>2<br>3 | `gform.addFilter(``'gform_chosen_options'``,``function``( options, element ) {`<br>```return``options;`<br>`} );` |

## Parameters

- **options** JS Object

The Chosen script options object. Additional documentation can be found in the [Chosen 1.1.0 release](https://github.com/harvesthq/chosen/releases/download/v1.1.0/chosen_v1.1.0.zip).

  - **allow\_single\_deselect** [bool](https://docs.gravityforms.com/bool)
    When set to true on a single select, Chosen adds a UI element which selects the first element (if it is blank).

  - **disable\_search** [bool](https://docs.gravityforms.com/bool)

    When set to true, Chosen will not display the search field (single selects only).

  - **disable\_search\_threshold** [integer](https://docs.gravityforms.com/integer)

    Hide the search input on single selects if there are fewer than (n) options.

  - **enable\_split\_word\_search** [bool](https://docs.gravityforms.com/bool)

    By default, searching will match on any word within an option tag. Set this option to false if you want to only match on the entire text of an option tag.

  - **inherit\_select\_classes** [bool](https://docs.gravityforms.com/bool)

    When set to true, Chosen will grab any classes on the original select field and add them to Chosen’s container div.

  - **max\_selected\_options** [integer](https://docs.gravityforms.com/integer)

    Limits how many options the user can select. When the limit is reached, the chosen:maxselected event is triggered.

  - **no\_results\_text** [string](https://docs.gravityforms.com/string)

    The text to be displayed when no matching results are found. The current search is shown at the end of the text (e.g., No results match “Bad Search”).

  - **placeholder\_text\_multiple** [string](https://docs.gravityforms.com/string)

    The text to be displayed as a placeholder when no options are selected for a multiple select.

  - **placeholder\_text\_single** [string](https://docs.gravityforms.com/string)

    The text to be displayed as a placeholder when no options are selected for a single select.

  - **search\_contains** [bool](https://docs.gravityforms.com/bool)

    By default, Chosen’s search matches starting at the beginning of a word. Setting this option to true allows matches starting from anywhere within a word. This is especially useful for options that include a lot of special characters or phrases in ()s and \[\]s.

  - **single\_backstroke\_delete** [bool](https://docs.gravityforms.com/bool)

    By default, pressing delete/backspace on multiple selects will remove a selected choice. When false, pressing delete/backspace will highlight the last choice, and a second press deselects it.

  - **width** [int](https://docs.gravityforms.com/int)

    The width of the Chosen select box. By default, Chosen attempts to match the width of the select box you are replacing. If your select is hidden when Chosen is instantiated, you must specify a width or the select will show up with a width of 0.

  - **display\_disabled\_options** [bool](https://docs.gravityforms.com/bool)

    By default, Chosen includes disabled options in search results with a special styling. Setting this option to false will hide disabled results and exclude them from searches.

  - **display\_selected\_options** [bool](https://docs.gravityforms.com/bool)

    By default, Chosen includes selected options in search results with a special styling. Setting this option to false will hide selected results and exclude them from searches.

Note: this is for multiple selects only. In single selects, the selected result will always be displayed.

- **element** [Js Object](https://docs.gravityforms.com/js-object)
The jQuery element object.


## Examples

### 1\. Limit the number of items which may be selected in a multi-select field.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5<br>6<br>7<br>8 | `gform.addFilter(``'gform_chosen_options'``,``function``( options, element ) {`<br>```//form id = 85, field id = 5`<br>```if``( element.attr(``'id'``) ==``'input_85_5'``) {`<br>```options.max_selected_options = 2;`<br>```}`<br>```return``options;`<br>`} );` |

### 2\. Configure Chosen to use a percentage based width.

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4<br>5 | `gform.addFilter(``'gform_chosen_options'``,``function``( options, element ) {`<br>```options.width =``'75%'``;`<br>```return``options;`<br>`} );` |

### 3\. Allow matches starting from anywhere within a word.

This can be done by enabling the search\_contains [option provided by the Chosen script](https://harvesthq.github.io/chosen/options.html).

|     |     |
| --- | --- |
| 1<br>2<br>3<br>4 | `gform.addFilter(``'gform_chosen_options'``,``function``( options, element ) {`<br>```options.search_contains =``true``;`<br>```return``options;`<br>`} );` |

JavaScript should be placed in a JS file included by your theme or a custom plugin.
