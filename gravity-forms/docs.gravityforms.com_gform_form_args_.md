## Description

The filter gform\_form\_args provides the ability to modify the options used to display the form.

## Usage

|     |
| --- |
| `add_filter(``'gform_form_args'``,``'your_function_name'``);` |

## Parameters

- **$form\_args** [array](https://docs.gravityforms.com/array)

An array of arguments used to customize the embed method for the form being displayed.

  - `form_id` [integer](https://docs.gravityforms.com/integer) \| [string](https://docs.gravityforms.com/string)

    The ID or title of the form to be displayed.
  - `display_title` [boolean](https://docs.gravityforms.com/boolean/)

    Indicates if the form title should be displayed.
  - `display_description` [boolean](https://docs.gravityforms.com/boolean/)

    Indicates if the form description should be displayed.
  - `force_display` [boolean](https://docs.gravityforms.com/boolean/)

    Indicates if the form should be displayed even if it is set to inactive.
  - `field_values` [array](https://docs.gravityforms.com/array)

    An array of dynamic population parameter keys with their corresponding values to be populated.
  - `ajax` [boolean](https://docs.gravityforms.com/boolean/)

    Indicates if the ifame-based legacy Ajax submission method is enabled.
  - `tabindex` [integer](https://docs.gravityforms.com/integer)

    `0` or the starting tabindex to be used when generating the input markup.
  - `submission_method` [null](https://docs.gravityforms.com/null) \| [string](https://docs.gravityforms.com/string)

    Since 2.9.0. Not set by default. The following are the possible string values when set, and the equivalent `GFFormDisplay` constants that can be used when setting the argument value:

    - `postback` (`GFFormDisplay::SUBMISSION_METHOD_POSTBACK`)

      Indicates the form should use the default postback submission method.
    - `iframe` (`GFFormDisplay::SUBMISSION_METHOD_IFRAME`)

      Indicates the form should use the iframe-based legacy Ajax submission method.
    - `ajax` (`GFFormDisplay::SUBMISSION_METHOD_AJAX`)

      Indicates the form should use the new Ajax submission method introduced in 2.9.0, which uses a true Ajax request. Some add-ons don’t currently support the new Ajax submission method, so you should only enable this feature on production sites for forms where you have fully tested it.

This code should be placed in the [functions.php](http://codex.wordpress.org/Theme_Development#Functions_File) file of your active theme.
